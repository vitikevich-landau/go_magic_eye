Что такое Око — за 90 секунд

go_magic_eye — учебная библиотека-«рентген»: она берёт живой объект Go и показывает его так, как его видит машина — байты, смещения, дыры выравнивания, заголовки строк и срезов, таблицы методов интерфейсов. Порт C++-проекта magic_eye, переосмысленный под механику Go.

eye.Inspect(&obj)   // статический осмотр: печать «рамки» в терминал
eye.Explore(&obj)   // странствие: интерактивный TUI-обозреватель
g := eye.NewGallery(); g.Add(&a).Add(&b).Run()

Конвейер

объект → [reflect+unsafe] → Model (plain-данные) → [render] → строки → печать
                               └────────→ [nav] дерево → [tui] кадры → терминал

Две границы («швы») держат систему простой: модель ничего не знает о рамках и цветах, вид ничего не знает о reflect. Проверяется списком import'ов.

Три идеи, на которых всё стоит

1. Рефлексия видит всё. reflect отдаёт имена, типы и offset'ы даже неэкспортированных полей; значение приватного поля читается легальным приёмом reflect.NewAt по живому адресу. Макросы-реестры C++-предка не нужны — это главный урок порта.
2. «Vtable» Go живёт в значении интерфейса. У объекта нет vptr. Пара слов (itab, data) — вся полиморфная машинерия; Око читает живой itab и называет методы через runtime.FuncForPC (§4).
3. Ленивость и честность. Дерево странствия строит узел только при раскрытии; всё, что нельзя показать корректно (nil, стёртый тип, значения map), называется по имени, а не замалчивается.

Память вживую

Так Око видит структуру. Наведи (или тапни) на байты:

type Knight struct {
    hp     int32   // приватное — Оку всё равно
    speed  int32
    armor  int16
    banner string
}
поле padding-дыра слова заголовка
Наведи на байт.
Каждый байт знает, зачем он: поле, дыра со своей причиной или слово заголовка строки.

Что здесь учат

  • Выравнивание: int16 после двух int32 оставляет дыру в 6 байт — string требует адрес, кратный 8. Go поля не переставляет: раскладка в руках автора.
  • little-endian: 100 = 64 00 00 00 — младший байт первым.
  • Заголовки: string — это 2 слова (data+len); сами буквы живут вне объекта и показываются «спутником».

Интерфейсы: два слова и таблица

Как Око читает itab

+0  inter *interfacetype   какой интерфейс
+8  _type *_type           какой динамический тип
+16 hash  uint32           копия хеша типа (ускоряет type switch)
+24 fun[0] uintptr ──→ runtime.FuncForPC → "main.(*Knight).Speak"
+32 fun[1] uintptr ──→ ...   (слоты отсортированы по имени метода)

Раскладка — внутренность gc-рантайма (стабильна go1.21+, но спецификация её не обещает). Око читает слоты только на 64-битных платформах и подписывает происхождение знания; имена методов из reflect верны всегда. Отдельная тонкость — direct-iface: для *T, map, chan, func и даже struct{p *T} слово data — само значение, а не адрес.

Структуры данных

Полные таблицы полей — в docs/data_model.md; здесь — роли и связи.

model.Model — всё, что Око увидело

Корень модели: Passport (статика типа), Regions (карта памяти), Embeds (встраивания), Ifaces (анатомия интерфейсов), Sats (память вне объекта), Bytes — живые байты через unsafe.Slice, не копия. Создатели: Of / OfValue / OfType. Единственный потребитель unsafe-знания.

model.Region — кусок памяти с выноской

Kind (поле/дыра/слово) + Offset,Size + готовые строки Value/Note/From. Инварианты: отсортированы, не пересекаются, каждая дыра несёт причину. Байты регион не хранит — рендер режет Model.Bytes.

model.Iface / Method — анатомия interface-значения

Два сырых слова (TabAddr/DataAddr), динамический тип, слоты (Name из reflect, PC из живого itab, Func из FuncForPC), флаг TypedNil.

nav.Node / nav.Session — граф странствия

Узел держит живой reflect.Value, ленивых детей (Kids() строит один раз), отказ Refusal, ссылку-цикл Cycle, пометку Copied. Session — плоский срез видимых узлов (дерево = список DFS по раскрытым), курсор, история прыжков и карта seen[{адрес,тип}], превращающая повторный адрес в узел ⟲.

tui.App / Decoder — экран и клавиши

Decoder — конечный автомат: рваные байты → клавиши (CSI/SS3/UTF-8; одинокий ESC дозревает по таймауту Flush). App — состояние кадра: фокус зон, панель (m/p/v/x), прокрутки, поиск, снимки; кадр перерисовывается только при изменениях, построчным диффом, в «синхронизированном выводе» (DECSET 2026).

render / text — вид и буквы

render превращает Model в []string с рамками (панели: всё/память/паспорт/интерфейсы/hex). text — единственная мера ширины (VisWidth учитывает ANSI и широкие руны), строитель Line, палитра и ASCII-fallback (Rune("▸",">")).

Взаимодействие за один кадр TUI

клавиша → Decoder → App.Handle → Session (курсор/раскрытие)
       → Node.Detail() [кэш] → model.OfValue → render.RenderPanel
       → App.Frame() (зоны) → дифф-draw → терминал

Странствие: цикл без горутин

for {
    if dirty { draw(дифф) }
    n := term.ReadInput(≤100 мс)   // termios VTIME · WaitForSingleObject
    if n==0 { Flush(ESC); resize() }
    for k := range keys { dirty = Handle(k) }
}
  • Почему без горутин-читателей: застрявший Read крал бы stdin у программы после выхода. Таймаут делает цикл живым: ESC дозревает, размер окна опрашивается.
  • Восстановление терминала — при любом исходе: нормальный выход — defer; Ctrl-C — сигнальная горутина лишь ставит флаг, цикл выходит сам и defer'ы отрабатывают в одном потоке (никаких гонок с draw).
  • Windows: та же логика; консоль переводится в VT-режим, очередь событий дренируется от фокусов/модификаторов, ввод — UTF-8.

Написать такое с нуля: дорожная карта

Этапы — реальная последовательность: каждый работает сам по себе и становится фундаментом следующего.

  1. M0 · паспорт. unsafe.Sizeof/Alignof, reflect.TypeOf, kind. Понять: тип — это метаданные, которые существуют в рантайме.
  2. M1 · hex-дамп. unsafe.Slice((*byte)(ptr), size) поверх любого объекта. Понять: адресуемость, «объект = байты», little-endian.
  3. M2 · поля и дыры. Обход NumField/Field(i).Offset, вычисление разрывов, причина каждой дыры = выравнивание следующего поля. Понять: раскладку структур и почему размер кратен align.
  4. M3 · приватное. flagRO и приём reflect.NewAt(t, unsafe.Pointer(v.UnsafeAddr())).Elem(). Понять: чем запрет reflect отличается от запрета памяти.
  5. M4 · заголовки. string/slice/map/chan/func как слова; unsafe.StringData, cap-хвост, спутники. Понять: значение ≠ данные, указатели повсюду.
  6. M5 · itab. Два слова интерфейса, раскладка internal/abi.ITab, runtime.FuncForPC, direct-iface. Понять: как type switch и вызов метода работают под капотом.
  7. M6 · терминал. termios (ECHO/ICANON/VMIN/VTIME) и консоль Windows (режимы, VT); ANSI: цвета, alt-screen, позиционирование; декодер ESC-последовательностей. Понять: терминал — это байтовый протокол двух направлений.
  8. M7 · странствие. Ленивое дерево поверх reflect.Value, плоский список видимых узлов, циклы по (адрес,тип), кадр-дифф. Понять: UI = чистая функция от состояния.

Ловушки и честные границы

ситуациячто делает Око
значения mapне адресуемы по правилам языка (бакеты переезжают) → честная копия с пометкой
unsafe.Pointer, uintptrтип стёрт / GC не охраняет → показывается слово, перехода нет
typed nilинтерфейс ≠ nil при nil-указателе внутри → подсвечено как ловушка
раскладка itab, hmapвнутренности рантайма: itab читается с подписью о происхождении, hmap — не читается вовсе
Inspect(x) по значениюосматривается коробка интерфейса (копия) — и это честно сказано; оригинал — через &x
жизнь объектовбайты и спутники — живая память: объекты галереи должны жить, пока идёт Run()
