# 👁 go_magic_eye (Око мага) — визуальный инспектор объектов Go

[![ci](https://github.com/vitikevich-landau/go_magic_eye/actions/workflows/ci.yml/badge.svg)](https://github.com/vitikevich-landau/go_magic_eye/actions/workflows/ci.yml)

Подключил один пакет — и смотришь внутрь значения. Паспорт типа, схема памяти
(поля с offset'ами, padding-дыры с причинами), заголовки `string`/`slice`/`map`,
анатомия interface-значений — **itab, «vtable» Go** — с живыми адресами методов,
встраивание структур (композиция вместо наследования) и панели-спутники для
памяти, живущей вне объекта.

Это порт учебной библиотеки [magic_eye](https://github.com/vitikevich-landau/magic_eye)
(C++) на Go — и одновременно урок о том, чем языки отличаются: рефлексию,
которую C++ ждал тридцать лет, Go носит с собой из коробки.

```go
import eye "github.com/vitikevich-landau/go_magic_eye"

eye.Inspect(&obj)             // полный осмотр по живому адресу (печать)
eye.Inspect(&obj, "казна")    // то же, со своей подписью в заголовке
eye.Inspect(obj)              // по значению: честный осмотр КОРОБКИ интерфейса
eye.InspectType[T]()          // только статика типа (объект не нужен)

eye.Explore(&obj)             // СТРАНСТВИЕ: интерактивный TUI-обозреватель
g := eye.NewGallery()         // несколько корней в одной сессии
g.Add(&knight, "рыцарь").Add(nums).AddType(eye.TypeOf[Config]())
g.Run()                       // блокирует до выхода (q/Esc)
```

## Что покажет

- **паспорт** — `Sizeof`/`Alignof`, kind, трейты: comparable или нет, видит ли
  тип GC («содержит указатели → GC сканирует»), нулевой размер;
- **встраивание** *(при анонимных полях)* — под-объекты встроенных типов: их
  offset, размер, продвинутые наружу методы; ромб встраивания честно даёт
  ДВЕ копии базы — virtual-баз в Go нет;
- **память** — главная секция: короткий итог (размер / поля / дыры), затем
  каждый регион (поле / padding / служебные слова) блоком строк с offset'ом,
  цветным «кирпичом», hex-байтами и ascii. Байт со смещением N стоит в колонке
  N%8 — выравнивание видно глазами. Сбоку выноски: имя/тип/значение поля
  (+hex и урок little-endian для целых), из какого встроенного типа поле
  пришло, причина каждой дыры («требует адрес, кратный 8»), совет по
  перестановке полей — Go, в отличие от компилятора, полей не переставляет;
- **интерфейсы** — блок-диаграмма «значение → два слова → itab → слот → код».
  Слоты `fun` читаются из живого itab, имена функций восстанавливаются через
  `runtime.FuncForPC`. Ловушка **typed nil** помечается прямо в дампе;
- **спутники** — панели после главной рамки: буфер строки (`.rodata` или
  куча), хребет среза с cap-хвостом, содержимое map, цель указателя.

Око само решает, что тип позволяет увидеть — и, в отличие от C++-предка,
макросы-реестры (`EYE_DESCRIBE`) ему не нужны:

```
reflect видит ВСЕ поля             ──►  имена, типы, offset'ы — даже неэкспортированные
unsafe читает по живому адресу     ──►  значения приватных полей (reflect.NewAt)
itab — внутренность gc-рантайма    ──►  читается только на 64-битных платформах,
                                        с честной пометкой происхождения знания
map-значения не адресуемы          ──►  честная копия с пометкой «копия»
unsafe.Pointer / nil               ──►  честный отказ: чужую память не читаем
```

## Странствие — интерактивный обозреватель (eye.Explore)

`eye.Explore(&obj)` открывает полноэкранный обозреватель в альтернативном
экране терминала: слева — дерево (встройки → поля → указатели → элементы
срезов страницами по 100 → пары map), справа — «Гримуар» выбранного узла,
внизу — гид по клавишам. Всё ленивое: узел разбирается только при раскрытии,
указатель разыменовывается только по вашей команде. После выхода терминал
восстанавливается (история прокрутки цела) — при любом исходе, включая
Ctrl-C и SIGTERM.

| клавиши | действие |
|---|---|
| `↑↓` (`k/j`) | курсор по дереву |
| `→`/`Enter` (`l`) | раскрыть узел · **перейти по указателю** |
| `←` (`h`) | свернуть · подняться к родителю |
| `g` / `⌫`, `b` | перейти по указателю / назад по истории |
| `Tab` | фокус: дерево ↔ детали (PgUp/PgDn листают фокусную зону) |
| `m` `p` `v` `x` | панель: память / паспорт / интерфейсы (vtable) / hex |
| `f` · `e` · `c` | развернуть длинные регионы · раскрыть ветку рекурсивно · свернуть всё |
| `1..9` | прыжок к N-му корню галереи |
| `/`, `n`, `N` | поиск по раскрытым узлам, следующее/предыдущее |
| `s` | **снимок экрана в файл** (`eye_snap_NNN.txt`, чистый текст без ANSI) |
| `?`/`F1` · `q`/`Esc` | помощь · выход |

Переходы по указателям типизированные: структуры, скаляры, срезы, map,
динамика интерфейсов. Честные отказы с причиной: `nil`, `unsafe.Pointer`
(тип стёрт), `func` (код, не данные). Указатель на уже показанный узел
(цикл, `self`) не плодит дублей — Око помечает его `⟲` и прыгает к
существующему узлу.

Контракт: объекты галереи и всё достижимое из них живут, пока идёт `Run()` —
Око смотрит на живую память и копий не делает (кроме значений map, которые
не адресуемы по правилам языка — такие узлы честно помечены).

Не-терминал (redirect в файл, CI) или `EYE_INTERACTIVE=0` — вместо TUI
печатается статический `Inspect` всех корней; программы со странствием
остаются пригодными для логов. `EYE_SCRIPT="down enter q"` — исполнить
клавиши из строки и напечатать кадры в stdout (так странствие покрыто
снапшот-тестами; см. `internal/tui/script_test.go`).

## Соответствие уроков: C++ ↔ Go

| C++ (magic_eye) | Go (go_magic_eye) |
|---|---|
| `sizeof` / `alignof`, трейты типов | `unsafe.Sizeof/Alignof`, `reflect.Kind`, comparable, GC-сканируемость |
| авторазбор агрегатов ≤8 полей | `reflect` разбирает ЛЮБУЮ структуру |
| `EYE_DESCRIBE`/`EYE_BASES` (рефлексии нет) | не нужны: reflect + `unsafe` видят всё |
| наследование, под-объекты баз | встраивание: под-объекты по offset'ам, продвижение методов |
| vptr В ОБЪЕКТЕ → vtable | объект чист; itab живёт В ЗНАЧЕНИИ ИНТЕРФЕЙСА |
| несколько vptr, offset-to-top | несколько itab на пары (тип, интерфейс); data всегда на начало объекта |
| ромб, общая virtual-база | ромб НЕ сливается: две честные копии базы |
| `std::string` (SSO/куча) | string-заголовок: data+len; `.rodata` против кучи; SSO нет |
| `std::vector` (data/size/cap) | срез: data+len+cap, cap-хвост под append |
| CRTP — статический полиморфизм | дженерики: мономорфизация, без слова-таблицы в объекте |
| «указатели трёх судеб» | живой / nil / `unsafe.Pointer` (тип стёрт) + `uintptr` вне защиты GC |
| — (аналога нет) | map (порядок нарочно случаен), каналы, `func`-замыкания, typed nil |

## Устройство

Организация — идиоматичная для Go: публичный пакет в корне модуля, вся
механика — в `internal/` (наружу не экспортируется), примеры — отдельные
`main`-пакеты, тесты — рядом с кодом.

Подробная документация — в `docs/`:

- [docs/architecture.md](docs/architecture.md) — карта пакетов, потоки
  Inspect и странствия (схемы), терминальный слой, как читается itab,
  соответствие C++-оригиналу, как расширять;
- [docs/data_model.md](docs/data_model.md) — все объекты системы и структуры
  данных: `Model`/`Region`/`Iface`/`Satellite`, `Node`/`Session`, `App`,
  жизненный цикл узла, сквозные инварианты.

```
go_magic_eye/
  eye.go            ← Inspect / InspectType / TypeOf (публичный API)
  explore.go        ← Explore / Gallery (странствие)
  options.go        ← переменные окружения EYE_*
  internal/
    model/          ← ШОВ №1: reflect+unsafe → плоские структуры данных
      model.go        типы модели (их и рисует вид)
      build.go        обход структур, дыры, встраивание
      leaf.go         поля по kind'ам: уроки little-endian, заголовков, ловушек
      iface.go        анатомия eface/iface, чтение itab, FuncForPC
      stringify.go    reflect.NewAt (чтение приватного), превью значений
    render/         ← ВИД: модель → строки (о reflect не знает)
      frame.go        рамки-картуши
      render.go       секции: паспорт, встраивание, заметки, спутники
      memory.go       карта памяти: кирпичи, hex, ascii, выноски
      iface_view.go   диаграмма «два слова → itab → слоты → код»
    term/           ← ОС-слой (ЕДИНСТВЕННЫЙ платформозависимый пакет)
      term_unix.go    termios (Linux/macOS) через syscall, без зависимостей
      term_windows.go консоль kernel32 + VT-режим (Windows 10+)
      term_fallback.go прочие ОС: честная деградация в печать
    nav/            ← навигационный граф странствия
      node.go         узел с ленивыми детьми
      build.go        типизированный строитель: отказы, циклы ⟲, страницы
      session.go      дерево, курсор, история, поиск
    tui/            ← ядро интерфейса (без внешних библиотек)
      keys.go         декодер клавиш: байты → события (чистый КА)
      app.go          цикл, диспетчер клавиш, снимки
      frame.go        кадр: зоны, дерево, «Гримуар», гид
      script.go       EYE_SCRIPT-режим (тесты, CI)
  examples/         ← 11 примеров (см. ниже), каждый самостоятелен
```

Граница «модель ↔ вид» держится списком import'ов: `render` не импортирует
`reflect`, `model` не знает про цвета и рамки. Зависимостей вне стандартной
библиотеки — ноль.

## Запуск

```bash
go run ./examples/06_interfaces          # статический осмотр (или странствие — см. ниже)
go run ./examples/11_explorer            # интерактивное странствие в терминале
go test ./...                            # тесты (модель, текст, клавиши, снапшоты TUI)
```

Нужен статический вывод для чтения/диффа — перенаправь в файл или пайп, Око
само деградирует в печать: `go run ./examples/11_explorer | less` (или
`EYE_INTERACTIVE=0 go run ./examples/11_explorer`).

## Переменные окружения

| переменная      | что делает                                              |
|-----------------|---------------------------------------------------------|
| `EYE_WIDTH=N`   | считать терминал N колонок (иначе определяем сами)       |
| `EYE_CENTER=0`  | не центрировать — прижать влево (для отчётов и диффов)   |
| `EYE_COLOR=1/0` | форсировать цвета вкл/выкл (CI, redirect в файл)         |
| `EYE_FULL=1`    | не сворачивать длинные регионы (`⋯ ещё N Б ⋯`)           |
| `EYE_INTERACTIVE=0` | `Explore` не входит в TUI — печатает как `Inspect`   |
| `EYE_SCRIPT="…"`| исполнить клавиши (`down enter q`) и печатать кадры      |
| `EYE_HEIGHT=N`  | высота кадра в EYE_SCRIPT-режиме (по умолчанию 40)       |
| `EYE_ASCII=1`   | рамки/стрелки — чистый ASCII (бедные шрифты)             |
| `EYE_SNAP_DIR=…`| каталог для снимков экрана клавишей `s` (иначе текущий)  |

Цвета и рамки включаются только в терминале: `go run ./... > log.txt` даст
чистый текст.

## Кроссплатформенность и честные границы

Go ≥ 1.22, без внешних зависимостей. Полноценные платформы: **Linux**,
**macOS** (termios через `syscall`), **Windows 10+** (консоль kernel32 +
VT-последовательности). Прочие ОС собираются и честно деградируют в
статическую печать.

- **Раскладка itab** (`inter/_type/hash/fun[]`) — внутренность gc-рантайма:
  стабильна в go1.21–go1.24, но спецификацией не обещана. Око читает слоты
  только на 64-битных платформах и всегда подписывает происхождение знания.
  Имена методов при этом честно берутся из `reflect` — они верны всегда.
- **Значения map** не адресуемы по правилам языка — узлы странствия для них
  являются копиями и помечены.
- **`Inspect(x)` по значению** осматривает коробку интерфейса (копию) — и
  говорит об этом; хочешь оригинал — передай `&x`. Сама упаковка в `any` —
  первый урок курса.
- **Чужая память не читается**: `unsafe.Pointer` (тип стёрт) и `uintptr`
  показываются как слова, переходов по ним нет.

## Родословная

Прямой предок — [magic_eye](https://github.com/vitikevich-landau/magic_eye):
C++-инспектор с теми же секциями, тем же странствием и теми же принципами
(модель отдельно от вида, ленивое дерево, честные отказы). Его `docs/legacy/`
рассказывает, как всё это строилось до эпохи рефлексии; эта библиотека —
рассказ о том, как та же задача выглядит в языке, где рефлексия есть с
рождения.

Промышленные родственники приёмов: `reflect` + `unsafe.Sizeof/Offsetof`
(вся механика), `runtime.FuncForPC` (имена методов), пакеты вроде
`davecgh/go-spew` и `dave/dst` (глубокая печать значений) — и
`internal/abi` самого рантайма, где раскладка itab описана первоисточником.
