# 👁 Playground — веб-версия Ока

Пишешь Go-код в браузере (редактор с настоящими диагностиками компилятора),
жмёшь «Взглянуть» — сервер собирает и запускает снипетт в песочнице, Око
отдаёт JSON-модель, фронтенд рисует её интерактивно: карта памяти с hover
по байтам, анимация little-endian, диаграмма itab, панели-спутники.

Подробное ТЗ и архитектура — [SPEC.md](SPEC.md).

## Запуск (Docker)

```bash
cd playground
docker compose up --build
# → http://localhost:8080
```

Один контейнер: собранный фронтенд вшит в бинарь сервера (`go:embed`),
внутри — Go toolchain (он компилирует твои снипетты; кеш сборки прогрет
при сборке образа, так что снипетт собирается ~за секунду).

## Разработка без Docker

```bash
# терминал 1: сервер API на :8080 (библиотеку найдёт сам, вверх от cwd)
cd playground/backend && go run .

# терминал 2: vite с горячей перезагрузкой на :5173 (проксирует /api)
cd playground/frontend && npm ci && npm run dev
```

Собрать фронт в прод-вид (ляжет в `backend/web/dist`, вшивается go:embed):

```bash
cd playground/frontend && npm run build
cd ../backend && go run .   # теперь отдаёт собранный фронт сам
```

## Как это устроено

- Код уходит в сборку **как есть**, одним `main.go` — тем же, что в
  `examples/` корня репозитория. Сервер запускает программу с
  `EYE_FORMAT=json`: Око печатает конверт с моделями вместо рамок, сервер
  выуживает конверты из stdout (печать пользователя уцелевает) и отдаёт
  фронту. Контракт конверта — [SPEC.md §2.1](SPEC.md).
- Песочница: `go build -gcflags=-e` (все ошибки, с позициями для маркеров
  редактора), `GOPROXY=off` — посторонние модули не собираются физически;
  запуск с таймаутом (по умолчанию 5 с), `GOMEMLIMIT`, обрезанием вывода
  и, где ОС позволяет, без сети (`unshare -r -n`).

## API

| метод и путь | вход | выход |
|---|---|---|
| `POST /api/check` | `{"code": "…"}` | `{ok, diagnostics: [{line, col, severity, message}]}` |
| `POST /api/run` | `{"code": "…"}` | `{ok, diagnostics, eye: <конверт>, stdout, stderr, timed_out, compile_ms, run_ms}` |
| `POST /api/explore` | `{"code": "…"}` | `{ok, session, roots, stdout, compile_ms}` — живой сеанс странствия |
| `POST /api/explore/cmd` | `{session, cmd: "kids"\|"detail", node}` | ответ протокола сеанса + свежий `stdout` |
| `POST /api/explore/close` | `{session}` | `{ok}` |
| `GET /api/examples` | — | `[{id, title, topic, code}]` |
| `GET /healthz` | — | `ok` |

**Странствие** — режим «странствие» в шапке: снипетт с `eye.Explore(&объект)`
не печатает, а остаётся жить — в браузере растёт ленивое дерево (поля,
переходы по указателям, честные отказы, циклы `⟲` и разделяемые ссылки `≡`
с прыжком к оригиналу), справа — Гримуар выбранного узла. Сеанс умирает по
закрытию вкладки, простою (3 мин) или возрасту (30 мин); одновременно живут
не больше 8 сеансов.

## Настройка сервера

| переменная | по умолчанию | что делает |
|---|---|---|
| `EYE_PG_ADDR` | `:8080` | адрес сервера |
| `EYE_PG_LIB` | поиск вверх от cwd | корень библиотеки Ока |
| `EYE_PG_RUN_TIMEOUT` | `5s` | потолок работы снипетта |
| `EYE_PG_COMPILE_TIMEOUT` | `30s` | потолок компиляции |
| `EYE_PG_ISOLATE` | `auto` | `auto`/`on`/`off` — сетевая изоляция запуска |

                                           # (нужен живой сервер на :8080)
